# V1.6 - 爆款素材库管理 - 产品实现计划

## 一、版本概述

**版本号**：V1.6  
**版本名称**：爆款素材库管理  
**目标**：用户可以收藏、整理、管理自己提取的爆款视频和文案，建立个人的爆款素材库，方便随时参考和检索。

**核心价值**：
- 解决用户"提取了很多爆款，但下次想找时找不到"的问题
- 帮助用户建立自己的爆款库，形成内容资产积累
- 通过标签和分类提升检索效率，让素材库真正可用

## 二、功能需求详细设计

### 2.1 收藏夹分类（P0 - 核心功能）

#### 功能描述
用户可以创建、编辑、删除收藏夹，将提取的爆款视频归类到不同收藏夹中。

#### 用户场景
- 用户提取了一个"美妆测评"类爆款视频，想保存到"美妆"收藏夹
- 用户想要创建新的收藏夹"美食探店"，并设置图标和描述
- 用户想要删除一个不需要的收藏夹

#### 详细需求

| 功能点 | 说明 | 交互方式 |
|-------|------|---------|
| **创建收藏夹** | 用户输入名称、可选图标/颜色、描述 | 点击"新建收藏夹"按钮，弹出表单 |
| **编辑收藏夹** | 修改名称、图标、描述 | 点击收藏夹的编辑按钮 |
| **删除收藏夹** | 删除收藏夹（可选：是否同时删除内部视频） | 点击删除按钮，二次确认 |
| **添加视频到收藏夹** | 将已提取的视频添加到一个或多个收藏夹 | 视频详情页点击"收藏"按钮，选择收藏夹 |
| **从收藏夹移除视频** | 从收藏夹中移除某条视频（不删除视频本身） | 收藏夹列表页点击"移除" |
| **收藏夹列表展示** | 显示所有收藏夹，包含名称、视频数量、创建时间 | 网格或列表布局 |
| **收藏夹详情页** | 展示收藏夹内所有视频列表，支持排序 | 点击收藏夹进入详情页 |

#### 数据模型设计

```sql
-- 收藏夹表
CREATE TABLE collections (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id TEXT NOT NULL,           -- 用户ID（数据隔离）
    name TEXT NOT NULL,              -- 收藏夹名称
    icon TEXT,                       -- 图标emoji或图标名称
    color TEXT,                      -- 颜色标记
    description TEXT,                -- 描述
    video_count INTEGER DEFAULT 0,   -- 视频数量（冗余字段，方便展示）
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id)
);

-- 收藏夹与视频关联表（多对多）
CREATE TABLE collection_videos (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    collection_id INTEGER NOT NULL,
    video_id TEXT NOT NULL,          -- 关联videos表的视频ID
    added_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(collection_id, video_id), -- 防止重复添加
    FOREIGN KEY (collection_id) REFERENCES collections(id) ON DELETE CASCADE,
    FOREIGN KEY (video_id) REFERENCES videos(id) ON DELETE CASCADE
);
```

#### API接口设计

```
POST   /api/collections              - 创建收藏夹
GET    /api/collections              - 获取当前用户所有收藏夹列表
GET    /api/collections/:id          - 获取单个收藏夹详情（含视频列表）
PUT    /api/collections/:id          - 更新收藏夹信息
DELETE /api/collections/:id          - 删除收藏夹

POST   /api/collections/:id/videos   - 添加视频到收藏夹
DELETE /api/collections/:id/videos/:video_id - 从收藏夹移除视频
```

### 2.2 标签打标（P0 - 核心功能）

#### 功能描述
用户可以为已提取的视频添加自定义标签，方便后续检索和分类。

#### 用户场景
- 用户给一个美妆视频打上"#口红测评"、"#国货"、"#平价"等标签
- 用户想要查看带"#美食"标签的所有视频
- 用户想要删除或修改某个标签

#### 详细需求

| 功能点 | 说明 | 交互方式 |
|-------|------|---------|
| **添加标签** | 为视频添加1个或多个标签 | 视频详情页输入标签，回车添加 |
| **删除标签** | 移除视频的某个标签 | 点击标签旁的"x"删除 |
| **标签建议** | 输入时自动提示已有的标签 | 输入标签时下拉提示 |
| **标签云/列表** | 展示用户所有使用过的标签及数量 | 标签管理页展示 |
| **按标签筛选** | 筛选包含特定标签的视频 | 标签列表页点击标签进入筛选结果 |

#### 数据模型设计

```sql
-- 标签表
CREATE TABLE tags (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id TEXT NOT NULL,           -- 用户ID
    name TEXT NOT NULL,              -- 标签名称
    usage_count INTEGER DEFAULT 0,   -- 使用次数
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(user_id, name),           -- 同一用户标签名唯一
    FOREIGN KEY (user_id) REFERENCES users(id)
);

-- 视频与标签关联表（多对多）
CREATE TABLE video_tags (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    video_id TEXT NOT NULL,
    tag_id INTEGER NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(video_id, tag_id),
    FOREIGN KEY (video_id) REFERENCES videos(id) ON DELETE CASCADE,
    FOREIGN KEY (tag_id) REFERENCES tags(id) ON DELETE CASCADE
);
```

#### API接口设计

```
POST   /api/tags                     - 创建/添加标签到视频
GET    /api/tags                     - 获取当前用户所有标签列表
DELETE /api/tags/:id                 - 删除标签（所有视频）
DELETE /api/tags/:id/video/:video_id - 移除视频的某个标签
GET    /api/tags/:id/videos          - 获取某标签下的所有视频
```

### 2.3 我的爆款库搜索（P0 - 核心功能）

#### 功能描述
用户可以通过关键词、标签、收藏夹等条件搜索自己收藏的爆款视频。

#### 用户场景
- 用户想要搜索包含"美妆"关键词的所有视频
- 用户想要查找同时带"#美食"和"#探店"标签的视频
- 用户想要在"美妆"收藏夹中搜索含"口红"的视频

#### 详细需求

| 功能点 | 说明 | 交互方式 |
|-------|------|---------|
| **关键词搜索** | 搜索视频标题、文案内容 | 搜索框输入关键词 |
| **标签筛选** | 按标签过滤视频 | 搜索页选择标签筛选 |
| **收藏夹筛选** | 按收藏夹过滤视频 | 搜索页选择收藏夹筛选 |
| **组合搜索** | 关键词+标签+收藏夹组合搜索 | 搜索页同时使用多个条件 |
| **搜索历史** | 保存最近搜索记录，方便快速重复搜索 | 搜索框下拉显示历史 |
| **结果排序** | 按时间、收藏数等排序 | 排序下拉菜单 |

#### API接口设计

```
GET    /api/videos/search            - 搜索视频（支持多参数）
       参数：?keyword=xxx&tag=xxx&collection_id=xxx&sort=time&limit=20&offset=0
GET    /api/search/history           - 获取搜索历史
POST   /api/search/history           - 保存搜索历史
DELETE /api/search/history           - 清空搜索历史
```

### 2.4 批量导出文案（P1 - 重要功能）

#### 功能描述
用户可以选择多个视频，批量导出文案内容到Markdown文件。

#### 用户场景
- 用户想要导出10个美妆测评视频的文案到本地备份
- 用户想要导出某个收藏夹的所有文案分享给团队
- 用户想要导出文案用于线下会议讨论

#### 详细需求

| 功能点 | 说明 | 交互方式 |
|-------|------|---------|
| **多选视频** | 在列表页勾选多个视频 | 列表页复选框选择 |
| **导出格式** | 导出为Markdown文件，包含标题、文案、标签等信息 | 点击"导出"按钮选择格式 |
| **导出范围** | 可选择仅导出文案，或包含分析结果 | 导出设置选项 |
| **按收藏夹导出** | 一键导出整个收藏夹 | 收藏夹详情页导出按钮 |

#### 导出Markdown格式示例

```markdown
# 爆款文案导出

- 标签：#美妆 #口红 #平价
- 收藏于：2024-01-15

### 文案内容
（这里放提取的文案）

---

## 视频2：美食探店 - 成都火锅
- 标签：#美食 #探店 #成都
- 收藏于：2024-01-16

### 文案内容
（这里放提取的文案）
```

#### API接口设计

```
POST   /api/export/markdown          - 批量导出文案到Markdown
       参数：{video_ids: ["id1", "id2", ...]}
```

### 2.5 结构统计分析（P2 - 增值功能）

#### 功能描述
统计分析用户收藏的爆款视频，分析哪种文案结构最常出现，帮助用户理解爆款规律。

#### 用户场景
- 用户想要了解自己收藏的爆款中，哪种开头钩子类型最多
- 用户想要知道收藏爆款的平均长度、结构分布
- 用户想要发现最常见的爆款标签组合

#### 详细需求

| 功能点 | 说明 | 交互方式 |
|-------|------|---------|
| **标签频率统计** | 显示最常用的标签TOP20 | 统计页展示词云或列表 |
| **文案长度分布** | 显示收藏爆款的长度区间分布 | 图表展示 |
| **收藏夹统计** | 显示各收藏夹的视频数量和占比 | 统计页展示 |
| **结构类型统计** | 如果有AI分析结果，统计开头钩子类型分布 | 图表展示 |

#### API接口设计

```
GET    /api/stats/tags               - 获取标签使用统计
GET    /api/stats/length-distribution - 获取文案长度分布
GET    /api/stats/collections        - 获取收藏夹统计
```

## 八、风险评估

| 风险 | 影响 | 应对措施 |
|-----|------|---------|
| 数据量大导致搜索慢 | 用户体验差 | SQLite添加索引优化，未来迁移到PostgreSQL |
| 标签重复/混乱 | 标签系统失效 | 提供标签建议，限制单个视频标签数量（如最多10个） |
| 用户不主动打标签 | 功能使用率低 | 提取完成后提示用户添加标签，提供一键添加AI推荐标签功能 |

## 附录：与V1.5的关联

V1.6的素材库管理依赖于V1.5的仿写系统：
- V1.5产出的仿写版本也可以保存到素材库
- 素材库可以区分"原始爆款"和"仿写版本"
- 搜索时可以筛选"仅原始"或"仅仿写"

建议在V1.5开发时就预留素材库相关数据模型，避免后续重构。
