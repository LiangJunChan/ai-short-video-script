# AI短视频脚本平台 - 产品路线图

## 一、产品定位重新梳理

### 1.1 目标用户
**专业编导、内容创作者、MCN机构、自媒体团队**

用户痛点：
- 刷到爆款视频想要参考文案，需要逐字听写，非常耗时
- 想要分析同行爆款，拆解文案结构，但没有高效工具
- 找到爆款视频后，需要快速提取文案并改写成自己的内容
- 需要发现当下热门选题，把握内容趋势

### 1.2 核心定位
**爆款短视频拆解工作台 - 输入视频链接 → 提取文案 → AI分析结构 → 快速仿写**

差异化对比：

| 平台 | 他们做什么 | 我们做什么 |
|------|-----------|-----------|
| 蝉妈妈/考古家 | 提供数据（播放、点赞、涨粉），找爆款选题 | 我们直接**拆解爆款内容本身**，拿到文案，分析结构，帮你快速仿写 |
| 剪映/剪映专业版 | 视频剪辑功能强大 | 我们专注**文案层面的拆解和再创作**，剪辑交给剪映，我们做剪映上游的工作 |
| ChatGPT | 通用AI创作，没有数据来源 | 我们从真实爆款视频提取文案，保证仿写出来的内容已经经过市场验证 |

### 1.3 当前已完成基础能力

| 功能 | 状态 | 说明 |
|-----|------|------|
| 视频上传 + 语音识别提取文案 | ✅ 完成 | 核心基础能力已有 |
| AI自定义改写文案 | ✅ 完成 | 支持用户给提示词改写 |
| 差异化积分扣费体系（提取5/改写10） | ✅ 完成 | 按使用付费模式就绪 |
| 每日签到获取积分 | ✅ 完成 | 用户留存机制 |
| 用户隔离 + 权限控制 | ✅ 完成 | 多用户数据隔离 |

## 二、竞品分析（蝉妈妈/考古家/灰豚数据）

### 2.1 这些平台的强项
- 强大的数据分析（流量走势、账号排名、品类数据）
- 爆款选题库、热榜合集
- 竞品账号监控
- 直播数据分析

### 2.2 这些平台的短板
- 只给你数据"哪条视频爆了"，不帮你拆解"为什么爆了"
- 不提供文案提取，需要你自己逐字听写
- 不帮你分析文案结构
- 不帮你改写成你自己的脚本

### 2.3 我们的机会
做**数据之后的那一步**：当编导从蝉妈妈找到爆款视频后，用我们工具一键提取文案、分析结构、AI仿写，直接输出可使用的新脚本。

| **爆款点提炼** | 🔴 P0 | AI分析这条视频爆在哪里（选题/表达方式/情绪价值） | 3积分/次 |
| **选题标签提取** | 🔴 P0 | 自动提取视频的核心选题和标签 | 2积分/次 |

**分析报告示例**：
```
【爆款文案结构分析】
- 开头钩子（0-3秒）："月薪3000和30000的区别在哪里？"
  特点：用对比引发好奇，直击痛点
- 主体内容（3-45秒）：分三点阐述不同思维方式
```

## 五、技术架构演进

### 当前架构
```
前端：React + Vite
后端：Go + Gin + SQLite
AI：本地Fun-ASR + 第三方LLM API
存储：本地文件
```

### 需要新增的能力

1. **视频链接解析模块** - 支持解析抖音/快手分享链接获取视频URL
2. **视频下载模块** - 后端下载视频到本地
3. **结构化存储** - 新增 `collections` `tags` 表支持收藏夹分类

### 未来演进

- SQLite → PostgreSQL（数据量大了之后）
- 本地文件 → 支持云存储OSS/COS
- Docker一键部署（方便用户私有化部署）

## 七、核心设计原则

1. **专注编导工作流**：从"找到爆款"到"写出新脚本"，只做这个流程内的事情，不碰视频剪辑
2. **提升效率**：帮编导节省听写和拆解的时间，原来1小时拆一个爆款，现在5分钟搞定
3. **隐私可控**：支持私有化部署，团队的爆款库数据自己掌握
4. **按使用付费**：不用不花钱，对中小团队友好
5. **AI增强**：AI用来分析和仿写，不是替你做决策，最终创意还是交给编导

---

## 八、总结

当前产品已经有了**视频上传+提取文案+AI改写**这个核心闭环。接下来沿着**"编导爆款拆解工作台"**这个方向：

1. 先做「链接一键提取」，解决用户最痛的"需要下载再上传"问题
2. 然后做「AI结构分析」，帮用户看懂爆款为什么爆
3. 再做「爆款仿写」，快速产出同结构新脚本
4. 然后做「素材库管理」，方便用户积累自己的爆款库
5. 逐步扩展到选题发现、团队协作

这样和蝉妈妈形成互补关系：用户用蝉妈妈找哪条视频爆了，用我们工具拆解仿写直接出脚本，我们做蝉妈妈没有的"文案层"能力，形成差异化。
